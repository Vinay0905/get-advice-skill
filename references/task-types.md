# Task Types

## Debugging
Use when the issue includes build failures, runtime errors, broken behavior, logs, stack traces, or repeated unsuccessful fixes.

Required senior output:
1. Likely root cause
2. Exact fix
3. Alternative fixes with tradeoffs
4. Risks or regressions
5. Tests to run
6. Next execution steps for Antigravity

## Architecture
Use when the issue involves system design, APIs, folder structure, modularization, scaling, data flow, or maintainability.

Required senior output:
1. Problem framing
2. Recommended design
3. Alternatives with tradeoffs
4. Implementation or migration steps
5. Pitfalls to avoid
6. Final recommendation

## Code Review
Use when the issue is about refactoring, readability, quality, maintainability, or best practices.

Required senior output:
1. Main issues
2. Concrete improvements
3. Patch or refactor guidance
4. Technical risks
5. Fix priority order

## Research
Use when the issue is about choosing tools, frameworks, APIs, or implementation approaches.

Required senior output:
1. Relevant options
2. Pros and cons
3. Best fit for this context
4. Reasoning
5. Adoption guidance

## Prompt Improvement
Use when the likely failure was caused by ambiguity, weak constraints, or poor task framing.

Required senior output:
1. Why the original request failed
2. Improved prompt
3. Stronger constraints
4. Acceptance criteria
5. Antigravity-optimized version
